Community Forum & Engagement Platform - Requirements Specification
Status: ACTIVE - Phase 2 Priority Layer: Community Engagement & Knowledge Sharing Dependencies: Economic Control System v1 (community bonuses), Content Library (knowledge base), Security System (moderation) Change Rate: Rapid (community-driven) Risk Level: Medium

Overview
Create a comprehensive community forum and engagement platform that enables users to ask questions, share knowledge, discuss platform features, and participate in governance discussions. The system rewards quality participation with Neuroni bonuses, implements robust moderation, builds user reputation through karma, and integrates with NOTA2 token governance for platform decision-making.

Architectural Principles
Integration Points:

Economic Control System: Triggers bonus awards for community participation
Content Library: Forum content feeds into searchable knowledge base
NOTA2 Token (Future): Governance proposals and voting discussions
Security System: Content moderation, spam prevention, abuse detection
Analytics System: Tracks engagement metrics and community health
Separation of Concerns:

Forum manages community interactions only
Economic Control handles bonus distribution
Content Library indexes valuable forum content
Security handles moderation and abuse
No direct modification of other systems
Problem Statement
The platform needs a community forum to:

Enable peer-to-peer support reducing support burden
Build community engagement and platform loyalty
Surface valuable insights from user discussions
Reward helpful community members with bonuses
Create space for governance discussions (NOTA2 future)
Moderate content to maintain quality and safety
Track user reputation and contributions
Foster knowledge sharing and collaboration
Jobs to Be Done
When I have a question about platform services, I want to ask the community and get helpful answers, So I can solve problems quickly without waiting for support.

When I know the answer to someone's question, I want to help them and earn recognition, So I can build reputation and earn Neuroni bonuses for my contributions.

When exploring platform features, I want to see discussions about use cases and best practices, So I can learn from other users' experiences and optimize my usage.

When governance decisions need community input (future), I want a dedicated space to discuss proposals and vote, So I can participate in platform direction with other token holders.

4. Reputation & Karma System
Karma Calculation:

// Calculate user karma score
function calculateKarma(user: User): number {
  const weights = {
    threadCreated: 5,
    replyPosted: 2,
    helpfulAnswer: 20,
    upvoteReceived: 1,
    downvoteReceived: -2,
    threadSolved: 15,
    moderatorAction: -50 // Penalty for violations
  }
  
  const karma = 
    (user.threadsCreated * weights.threadCreated) +
    (user.repliesPosted * weights.replyPosted) +
    (user.helpfulAnswers * weights.helpfulAnswer) +
    (user.upvotesReceived * weights.upvoteReceived) +
    (user.downvotesReceived * weights.downvoteReceived) +
    (user.threadsSolved * weights.threadSolved) +
    (user.moderatorActions * weights.moderatorAction)
  
  return Math.max(karma, 0) // Never negative
}

Acceptance Criteria
Thread Creation
Given a user creates a thread with valid content, When thread is submitted, Then thread is published immediately, appears in category, and user is notified of quality gate requirement.

Given a thread receives 3 upvotes within 24 hours, When quality gate is met, Then 15 Neuroni bonus is awarded to author and transaction is logged.

Given a thread does not meet quality gate, When 24 hours pass, Then no bonus is awarded but thread remains published.

Helpful Answer
Given a thread author marks a reply as helpful, When helpful marking is confirmed, Then reply is highlighted, 10 Neuroni bonus is awarded to replier, 20 karma is added, and thread status changes to "Solved".

Given a reply is marked helpful, When bonus is awarded, Then replier receives notification with bonus confirmation and updated balance.

Voting
Given a user upvotes a post, When vote is cast, Then vote is recorded, post score increases, author gains 1 karma, and vote is saved.

Given a user tries to vote on their own post, When vote is attempted, Then vote is rejected with message "Cannot vote on your own posts".

Given a reply reaches 5 upvotes, When 5th upvote is cast, Then 5 Neuroni quality bonus is awarded to author and notification is sent.

Search & Discovery
Given a user searches forum content, When search query is entered, Then relevant threads and replies are returned ranked by relevance within 2 seconds.

Given a user filters by category and status, When filters are applied, Then results update to show only matching threads.

Notifications
Given a user receives a reply to their thread, When reply is posted, Then user receives in-app and email notification (based on preferences) within 30 seconds.

Given a user is mentioned in a reply, When @mention is posted, Then mentioned user receives notification with link to reply.

Moderation
Given a user reports inappropriate content, When report is submitted, Then content is flagged, moderator is notified, and reporter receives confirmation.

Given a moderator reviews flagged content, When moderation action is taken, Then content is removed/edited/approved, author is notified, and action is logged.

Reputation
Given a user earns karma through activities, When karma threshold is reached, Then reputation level is upgraded, user is notified, and new permissions are granted.

Given a user views leaderboard, When leaderboard loads, Then top contributors are displayed with karma scores, helpful answers, and reputation levels.

Error Handling
Given bonus distribution fails, When Economic Control returns error, Then helpful marking is preserved, bonus is queued for retry, and admin is alerted.

Given notification service is unavailable, When notification should be sent, Then notification is queued for retry and delivered when service recovers.
