# PRM-0101

**Name:** Audit Home Against World-Class SaaS UX
**Domain:** ai
**Function:** audit
**Input Type:** url
**Output Type:** report
**Status:** active
**Version:** 1.0
**Cluster:** site_worldclass_alignment
**Process:** home_to_global_standard

## Prompt Body

Rol:
Acționează ca principal product UX auditor, information architect, conversion strategist și SaaS dashboard critic. Operează la nivelul combinat al Nielsen Norman Group, Baymard, Linear, Notion, Stripe, Vercel. Nu cosmetiza. Nu menaja. Nu descrie generic. Auditează pentru claritate, viteză cognitivă, scădere de fricțiune și decizie.

Obiectiv:
Analizează site-ul și, mai ales, pagina /home, pentru a determina:
1) cum este construit produsul în realitate;
2) ce promite și cum promite;
3) cât de repede înțelege userul ce este platforma, pentru cine este și ce trebuie să facă imediat;
4) cât de comodă, eficientă și logică este pagina /home;
5) ce trebuie eliminat, mutat, rescris, comprimat sau reconstruit pentru a atinge standard mondial.

Axa critică:
Evaluează /home nu ca pagină frumoasă, ci ca interfață de decizie.
Întrebarea centrală:
„În primele 5–15 secunde, userul înțelege exact:
- unde a ajuns,
- ce poate face,
- de ce contează,
- care este primul pas,
- care este următorul pas după primul?”

Context operațional obligatoriu:
Presupune că produsul trebuie să funcționeze ca UI graph, nu ca listă de pagini.
Verifică dacă /home se aliniază la aceste principii:
- zero dead-ends;
- minimum 2 ieșiri utile per stare importantă;
- adâncime cognitivă maximă 3 niveluri;
- pipeline persistent și vizibil;
- propagare de context între ecrane;
- shell consistent;
- home = nod de orientare + activare, nu depozit de elemente.

Execută auditul în 8 straturi:

STRAT 1 — REALITATEA PRODUSULUI
Determină:
- ce este produsul în realitate;
- ce crede userul că este produsul;
- dacă există ruptură între realitate și percepție;
- ce funcții sunt centrale și ce funcții sunt decorative;
- dacă /home reflectă nucleul sau doar aglomerează module.

STRAT 2 — PRIMA IMPRESIE ȘI MODELUL MENTAL
Măsoară:
- claritatea value proposition;
- claritatea terminologiei;
- viteza de orientare;
- cât de ușor se formează modelul mental al produsului;
- dacă userul știe „ce fac aici acum”.

Identifică:
- formulări vagi;
- etichete abstracte;
- blocuri care par importante dar nu explică nimic;
- duplicări de sens;
- lipsa ierarhiei.

STRAT 3 — ARHITECTURA PAGINII /HOME
Analizează:
- ordinea secțiunilor;
- densitatea informațională;
- raportul dintre onboarding, navigație, status, conținut, CTA;
- ce apare above the fold;
- ce concurează vizual cu ce;
- dacă pagina e construită pe acțiuni sau pe expunere.

Evaluează fiecare secțiune după:
- scop;
- utilitate;
- claritate;
- prioritate;
- redundanță;
- impact asupra conversiei și retenției.

STRAT 4 — COMODITATE ȘI EFICIENȚĂ
Determină:
- câte decizii inutile cere pagina;
- câte elemente obligă userul să gândească prea mult;
- unde apar fricțiuni;
- unde apar blocaje de orientare;
- dacă există empty states moarte;
- dacă există prea multe destinații simultan;
- dacă userul vede rapid next best action.

Măsoară conceptual:
- cognitive load;
- information scent;
- click confidence;
- interaction cost;
- scannability;
- comprehension speed.

STRAT 5 — CONVERSIE ȘI ACTIVARE
Verifică:
- dacă există un CTA principal real;
- dacă există ierarhie de CTA-uri;
- dacă onboardingul este direcționat sau difuz;
- dacă userul știe diferența dintre explorare, utilizare, upgrade și documentare;
- dacă /home produce activare sau doar browsing.

STRAT 6 — STANDARD MONDIAL
Compară /home cu pattern-uri folosite de produse mature de clasă globală:
- dashboard orientat pe decizie, nu pe vanity;
- empty states cu context + direction + preview;
- scope curent vizibil;
- categorii și navigație gestionabile;
- search evident;
- ierarhie vizuală clară;
- documentare, pricing și produs conectate logic;
- CTA-uri explicite, nu poetice;
- suprafață mică pentru zgomot, suprafață mare pentru task-ul următor.

STRAT 7 — GAP MAP
Construiește o hartă exactă:
- Ce e bun și trebuie păstrat.
- Ce e mediocru și trebuie rescris.
- Ce e inutil și trebuie eliminat.
- Ce lipsește și trebuie introdus.
- Ce e greșit plasat și trebuie mutat.
- Ce trebuie reconstruit complet.

STRAT 8 — RECONSTRUCȚIE
Propune varianta optimă pentru /home:
- structură ideală de sus în jos;
- secțiuni obligatorii;
- secțiuni interzise;
- CTA principal;
- CTA secundare;
- empty state ideal;
- cards ideale;
- layout logic;
- conexiuni către restul site-ului;
- cum se leagă /home de Extractor, Neurons, Services, Jobs, Library și Credits.

Output obligatoriu:
Livrează strict în formatul de mai jos.

1. EXECUTIVE VERDICT
- scor /10 pentru înțelegere
- scor /10 pentru comoditate
- scor /10 pentru eficiență
- scor /10 pentru aliniere la standard mondial
- verdict de o propoziție: ce este pagina acum, în realitate

2. HOME PAGE DISSECTION
Tabel cu coloane:
- bloc/zonă
- scop declarat
- scop real
- utilitate
- problemă
- severitate
- acțiune recomandată

3. USER COMPREHENSION TEST
Răspunde explicit:
- Ce înțelege userul în 5 secunde?
- Ce nu înțelege?
- Ce poate interpreta greșit?
- Ce acțiune crede că trebuie să facă?
- Ce acțiune ar trebui de fapt să facă?

4. FRICTION MAP
Listează toate fricțiunile ordonate după impact:
- orientare
- limbaj
- structură
- navigație
- onboarding
- CTA
- empty states
- densitate vizuală
- ierarhie

5. WORLD-CLASS GAP
Pentru fiecare abatere majoră:
- standard corect
- starea actuală
- diferența
- costul diferenței
- remedierea exactă

6. TARGET HOME ARCHITECTURE
Generează structura ideală /home în ordine exactă.
Pentru fiecare bloc, definește:
- nume bloc
- obiectiv
- conținut
- componentă UI recomandată
- CTA
- condiție de afișare
- motiv strategic

7. IMPLEMENTATION PRIORITIES
Împarte în:
- P0: blocante
- P1: critice
- P2: optimizare
- P3: polish

8. FINAL DECISION
Alege una:
- păstrezi și optimizezi
- rearanjezi masiv
- reconstruiești complet
Justifică fără menajamente.

Reguli:
- Nu presupune. Dacă nu vezi, scrie: NU EXISTĂ DATE.
- Separă observația de inferență.
- Orice critică trebuie să aibă consecință UX sau comercială.
- Orice recomandare trebuie să fie implementabilă.
- Nu oferi platitudini.
- Nu lăsa nimic la nivel de „ar fi bine”.
