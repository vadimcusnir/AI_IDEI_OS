# Comprehensive Gamification System

## Overview

Implement a sophisticated gamification layer across the AI-IDEI platform that drives retention, educates users on product capabilities, and creates meaningful identity within the creator economy. The system builds upon the existing `useXP` hook foundation while adding competitive elements, achievement complexity, and psychological engagement mechanics.

## Problem Statement

The platform currently has a basic XP system (levels, badges, streaks) but lacks:
- **Retention mechanisms**: No compelling reason for daily/weekly return beyond core functionality
- **Skill scaffolding**: Users don't understand the full capability spectrum or progression path
- **Social proof**: No competitive or collaborative elements to drive engagement
- **Identity formation**: Users can't express mastery or build reputation within the ecosystem

This creates churn risk and underutilization of advanced features, particularly critical for one-person businesses and creators who need sustained engagement to extract value.

## Strategic Goals

### Primary: Retention & Habit Formation (Behavioral Control)
- Increase DAU/WAU through daily/weekly usage loops
- Reward consistency over intensity (streak mechanics with soft decay)
- Make inactivity visible without punitive hard resets
- Validate product-market fit through engagement metrics

### Secondary: Skill Progression & User Education (Cognitive Scaffolding)
- Unlock complexity gradually through XP-gated features
- Signal readiness for advanced capabilities
- Map actions to competence, not just time spent
- Reduce misuse and support load through behavioral training

### Tertiary: Motivation & Identity (Status Layer)
- Create visible identity through named ranks and capability badges
- Support future community dynamics
- Build long-term brand affinity

## Core Principles

### Economic Separation (Non-Negotiable)
**Rule Zero**: Soft economy (XP) must never mint hard economy (neurons/credits).

- **XP ≠ Currency**: XP is a control signal, not value
- **XP → Entitlement Modifiers**: Affects how credits are used, not balance
- **XP as Trust Signal**: Modulates system trust and access levels
- **Badges as Capability Flags**: Boolean unlocks, not point multipliers
- **Streaks Affect Priority**: Queue priority, not quantity

**Mental Model**: Credits = fuel, XP = license class. You need fuel to drive; XP determines which vehicles you can operate.

## User Touchpoints to Gamify

### High-Leverage Touchpoints (Core)

#### 1. First Successful Value Delivery
**Trigger**: First completed neuron output, first artifact exported, first deliverable generated

**Why**: Anchors perceived value and sets baseline expectancy

**Gamification**:
- One-time XP spike: +100 XP
- "Activation" badge (non-repeatable): "First Steps"
- Celebratory notification with next milestone preview

#### 2. Correct/High-Quality Usage
**Trigger**: Using advanced features correctly, supplying clean inputs, low retry/error patterns

**Why**: Trains behavior (not volume), reduces system cost and support load

**Gamification**:
- XP weighted by quality: Base XP × quality multiplier (1.0-2.0)
- Silent multipliers (no fireworks, subtle UI indicator)
- Quality badges: "Precision Operator", "Clean Input Master"

#### 3. Consistency Over Time (Return Cycles)
**Trigger**: Daily/weekly active return, completing multi-step flows, maintaining operational cadence

**Why**: Predicts retention better than raw usage, stabilizes demand

**Gamification**:
- Streak system with soft decay (3-day grace period)
- Small, capped daily XP: +10 XP base, +5 XP per streak day (max +50)
- Streak milestone badges: 7-day, 30-day, 90-day, 365-day

### Medium-Leverage Touchpoints (Selective)

#### 4. Progression Milestones
**Trigger**: Unlocking higher-risk actions, new modes, reduced friction paths

**Why**: Converts learning into entitlement, makes XP operational

**Gamification**:
- Level-based unlocks (see Level System below)
- Explicit capability badges: "Advanced Operator", "Power User"
- Unlock notifications with feature tutorials

#### 5. Resilience & Recovery
**Trigger**: Successfully fixing failed jobs, recovering from errors, adapting after rejection

**Why**: Teaches system boundaries, rewards maturity over perfection

**Gamification**:
- XP on successful recovery: +25 XP
- Invisible trust score increments (affects future friction)
- Resilience badges: "Problem Solver", "Persistent Creator"

## XP & Level System

### XP Calculation Formula
```
Level N requires: N² × 100 XP total
Level 1: 0 XP
Level 2: 100 XP
Level 3: 400 XP
Level 4: 900 XP
Level 5: 1,600 XP
...
Level 10: 10,000 XP
```

### Level Progression & Unlocks

| Level   | XP Required   | Named Rank   | Unlocks                                       |
| ------- | ------------- | ------------ | --------------------------------------------- |
| 1       | 0             | Novice       | Basic features, 1 concurrent job              |
| 2       | 100           | Apprentice   | Batch processing (3 items)                    |
| 3       | 400           | Creator      | Custom templates, 2 concurrent jobs           |
| 5       | 1,600         | Artisan      | Advanced services, priority queue             |
| 7       | 4,900         | Expert       | Reduced cooldowns, 5 concurrent jobs          |
| 10      | 10,000        | Master       | All features, 10 concurrent jobs, beta access |
| 15      | 22,500        | Virtuoso     | Custom integrations, API access               |
| 20      | 40,000        | Legend       | White-label options, dedicated support        |

### Daily XP Cap
**Base cap**: 200 XP per day (prevents farming, encourages consistency)

**Exceptions**:
- First-time achievements: No cap
- Weekly challenges: Bonus XP outside cap
- Quality multipliers: Apply before cap calculation

## Streak System

### Streak Mechanics
- **Activation**: Any meaningful action (neuron creation, deliverable generation, service usage)
- **Daily window**: 24-hour rolling window from last action
- **Grace period**: 3-day soft decay (streak preserved but flagged)
- **Decay notification**: Warning at 48h of inactivity
- **Soft reset**: After 3 days, streak resets but "longest streak" preserved

### Streak Rewards
- **Daily XP bonus**: +5 XP per streak day (capped at +50)
- **Milestone badges**: 7, 30, 90, 365 days
- **Priority queue**: 30+ day streaks get queue priority
- **Streak recovery**: One "Streak Freeze" token per month (preserves streak for 24h)

## Technical Requirements

### Backend (Supabase)

#### Database Functions

**award_xp** (Enhanced)
```sql
CREATE OR REPLACE FUNCTION award_xp(
  p_user_id UUID,
  p_event_type TEXT,
  p_xp_amount INTEGER,
  p_quality_multiplier NUMERIC DEFAULT 1.0,
  p_metadata JSONB DEFAULT '{}'
) RETURNS TABLE (
  new_xp INTEGER,
  new_level INTEGER,
  level_up BOOLEAN,
  achievements_unlocked TEXT[]
) AS $$
DECLARE
  v_current_xp INTEGER;
  v_current_level INTEGER;
  v_new_xp INTEGER;
  v_new_level INTEGER;
  v_daily_xp INTEGER;
  v_xp_cap INTEGER := 200;
  v_achievements TEXT[];
BEGIN
  -- Get current state
  SELECT total_xp, current_level INTO v_current_xp, v_current_level
  FROM user_levels
  WHERE user_id = p_user_id
  FOR UPDATE;

  -- Check daily XP cap
  SELECT COALESCE(SUM(xp_amount), 0) INTO v_daily_xp
  FROM xp_events
  WHERE user_id = p_user_id
    AND created_at >= CURRENT_DATE;

  -- Apply cap (unless first-time achievement)
  IF v_daily_xp >= v_xp_cap AND p_event_type NOT LIKE 'achievement.%' THEN
    RETURN QUERY SELECT v_current_xp, v_current_level, FALSE, ARRAY[]::TEXT[];
    RETURN;
  END IF;

  -- Calculate XP with multiplier
  v_new_xp := v_current_xp + FLOOR(p_xp_amount * p_quality_multiplier);

  -- Calculate new level
  v_new_level := FLOOR(SQRT(v_new_xp / 100));

  -- Update user_levels
  UPDATE user_levels
  SET total_xp = v_new_xp,
      current_level = v_new_level,
      last_action_at = NOW(),
      grace_period_active = FALSE,
      updated_at = NOW()
  WHERE user_id = p_user_id;

  -- Insert XP event
  INSERT INTO xp_events (user_id, event_type, xp_amount, quality_multiplier, metadata)
  VALUES (p_user_id, p_event_type, p_xp_amount, p_quality_multiplier, p_metadata);

  -- Check for new achievements
  v_achievements := check_achievements(p_user_id);

  -- Return results
  RETURN QUERY SELECT v_new_xp, v_new_level, (v_new_level > v_current_level), v_achievements;
END;
$$ LANGUAGE plpgsql;
```

**update_streak** (New)
```sql
CREATE OR REPLACE FUNCTION update_streak(
  p_user_id UUID
) RETURNS TABLE (
  current_streak INTEGER,
  grace_period_active BOOLEAN,
  streak_broken BOOLEAN
) AS $$
DECLARE
  v_last_action TIMESTAMP;
  v_current_streak INTEGER;
  v_longest_streak INTEGER;
  v_grace_active BOOLEAN;
  v_hours_since_action INTEGER;
BEGIN
  -- Get current state
  SELECT last_action_at, streak_days, longest_streak, grace_period_active
  INTO v_last_action, v_current_streak, v_longest_streak, v_grace_active
  FROM user_levels
  WHERE user_id = p_user_id
  FOR UPDATE;

  -- Calculate hours since last action
  v_hours_since_action := EXTRACT(EPOCH FROM (NOW() - v_last_action)) / 3600;

  -- Streak logic
  IF v_hours_since_action <= 24 THEN
    -- Within 24h: increment streak
    v_current_streak := v_current_streak + 1;
    v_grace_active := FALSE;
    
    -- Update longest streak
    IF v_current_streak > v_longest_streak THEN
      v_longest_streak := v_current_streak;
    END IF;

    UPDATE user_levels
    SET streak_days = v_current_streak,
        longest_streak = v_longest_streak,
        grace_period_active = FALSE,
        last_action_at = NOW()
    WHERE user_id = p_user_id;

    RETURN QUERY SELECT v_current_streak, FALSE, FALSE;

  ELSIF v_hours_since_action <= 72 THEN
    -- 24-72h: grace period
    IF NOT v_grace_active THEN
      UPDATE user_levels
      SET grace_period_active = TRUE
      WHERE user_id = p_user_id;
    END IF;

    RETURN QUERY SELECT v_current_streak, TRUE, FALSE;

  ELSE
    -- >72h: streak broken
    UPDATE user_levels
    SET streak_days = 0,
        grace_period_active = FALSE
    WHERE user_id = p_user_id;

    RETURN QUERY SELECT 0, FALSE, TRUE;
  END IF;
END;
$$ LANGUAGE plpgsql;
```

**check_achievements** (New)
```sql
CREATE OR REPLACE FUNCTION check_achievements(
  p_user_id UUID
) RETURNS TEXT[] AS $$
DECLARE
  v_new_achievements TEXT[] := ARRAY[]::TEXT[];
  v_achievement RECORD;
BEGIN
  -- Check each achievement's requirements
  FOR v_achievement IN 
    SELECT a.id, a.requirements
    FROM achievements a
    WHERE a.id NOT IN (
      SELECT achievement_id FROM user_achievements WHERE user_id = p_user_id
    )
  LOOP
    -- Evaluate requirements (simplified - actual logic would be more complex)
    IF evaluate_achievement_requirements(p_user_id, v_achievement.requirements) THEN
      -- Unlock achievement
      INSERT INTO user_achievements (user_id, achievement_id)
      VALUES (p_user_id, v_achievement.id);

      -- Add to badges array
      UPDATE user_levels
      SET badges = array_append(badges, v_achievement.id)
      WHERE user_id = p_user_id;

      v_new_achievements := array_append(v_new_achievements, v_achievement.id);
    END IF;
  END LOOP;

  RETURN v_new_achievements;
END;
$$ LANGUAGE plpgsql;
```
