# Security Hardening — WAVE 0 (F-002, F-004, F-010)

## ✅ F-002 — Free unlimited AI via chat endpoints (FIXED)

**Status**: Sealed.

`neuron-chat` and `notebook-chat` now atomically deduct NEURONS via
`public.atomic_deduct_neurons()` before calling the AI gateway.

- `neuron-chat`: 5 NEURONS / message
- `notebook-chat`: 7 NEURONS / message

If the AI gateway returns 4xx/5xx, we refund via `public.add_credits()`.
Validation failures also refund.

**Verify**:
```bash
# Without credits → 402
curl -X POST https://<project>.functions.supabase.co/neuron-chat \
  -H "Authorization: Bearer <jwt-with-zero-balance>" \
  -d '{"messages":[{"role":"user","content":"hi"}]}'
# Expect: {"error":"AI credits exhausted...","code":"INSUFFICIENT_CREDITS","required":5}
```

## ⚠️ F-004 — `.env` tracked in git (PARTIAL)

**Status**: Mitigated, requires manual git history scrub.

The committed `.env` contains **only**:
- `SUPABASE_URL` (public, in client bundle)
- `SUPABASE_PUBLISHABLE_KEY` (public anon key — designed for client exposure)
- `VITE_*` mirrors of the same

**No private keys leaked.** However, tracking `.env` is a bad pattern.

### Manual remediation required

Lovable's git workflow disallows direct git commands from the agent.
The user must run, locally:

```bash
# 1. Remove from index, keep on disk
git rm --cached .env
echo ".env" >> .gitignore
echo ".env.*" >> .gitignore
echo "!.env.example" >> .gitignore
git commit -m "chore(security): untrack .env (only contained publishable keys)"

# 2. (Optional) scrub from history — only needed if private keys ever passed through
# git filter-repo --path .env --invert-paths
# git push --force-with-lease origin main
```

`.env` is auto-regenerated by Lovable Cloud on every preview build,
so removing it from tracking is non-disruptive.
